1. Java 的三大特性是什么？

答案：

· 封装：将对象的属性和实现细节隐藏，仅通过公共方法对外提供访问，提高安全性和易用性。
· 继承：子类可以复用父类的属性和方法，并可以扩展或重写，实现代码复用。
· 多态：同一个行为具有多个不同表现形式，例如接口的多个不同实现，或父类引用指向子类对象。

2. String、StringBuffer、StringBuilder 的区别？

答案：

· String：不可变，每次修改（如拼接）都会创建新对象，适合少量字符串操作。
· StringBuffer：可变，线程安全（方法使用 synchronized 修饰），效率相对较低。
· StringBuilder：可变，非线程安全，效率最高，适合单线程下的大量字符串操作。

3. == 和 equals() 的区别？

答案：

· == 比较基本数据类型时比较值，比较引用类型时比较内存地址（是否为同一对象）。
· equals() 是 Object 类的方法，默认行为与 == 相同，但常被重写（如 String、Integer）用于比较对象内容是否相等。

4. final 关键字的作用？

答案：

· 修饰类：该类不能被继承。
· 修饰方法：方法不能被重写。
· 修饰变量：基本类型变量值不能改变，引用类型变量不能指向其他对象，但对象内部属性可变。

5. static 关键字的作用？

答案：

· 修饰成员变量：成为类变量，所有实例共享一份。
· 修饰方法：成为类方法，可直接通过类名调用，不能访问非静态成员。
· 静态代码块：类加载时执行一次，常用于初始化静态资源。
· 静态内部类：不需要依赖外部类实例。

6. Java 的异常体系是怎样的？

答案：

· 根类是 Throwable，下分 Error 和 Exception。
· Error 表示系统级错误（如 OutOfMemoryError），程序无法处理。
· Exception 分为受检异常（如 IOException，编译时强制处理）和非受检异常（如 NullPointerException，运行时异常，不强制处理）。

7. 什么是反射？

答案：
反射允许程序在运行时获取任意类的信息（构造器、方法、字段），并能动态创建对象、调用方法、修改属性。常用于框架开发（如 Spring）、动态代理等。示例：Class.forName("com.example.User").newInstance()。

8. 什么是注解？

答案：
注解是一种元数据，可附加在类、方法、字段等元素上，用于提供配置信息或标记。常见内置注解：@Override、@Deprecated、@SuppressWarnings。自定义注解使用 @interface 定义，可结合反射在运行时处理。

9. JVM 的内存结构是怎样的？

答案：

· 堆（Heap）：存放对象实例，所有线程共享，垃圾回收主要区域。
· 方法区（Method Area）：存储类信息、常量、静态变量、JIT 编译后的代码（JDK8 后元空间取代永久代）。
· 虚拟机栈（VM Stack）：每个线程私有，存放局部变量表、操作数栈、方法出口等。
· 本地方法栈（Native Method Stack）：为 native 方法服务。
· 程序计数器（PC Register）：记录当前线程执行的字节码行号。

10. Java 的垃圾回收机制（GC）是什么？

答案：
GC 自动回收不再使用的对象内存。判断对象是否“存活”的依据是可达性分析（GC Roots 作为起点）。常见回收算法：标记-清除、复制、标记-整理。HotSpot 分代收集：新生代（Eden + Survivor）使用复制算法，老年代使用标记-整理或标记-清除。

11. 接口和抽象类的区别？

答案：

· 抽象类使用 abstract 关键字，接口使用 interface。
· 抽象类可以有构造器、成员变量、非抽象方法；接口在 JDK8 前只能有常量和抽象方法，JDK8 后允许默认方法和静态方法。
· 类只能单继承，但可以多实现接口。
· 抽象类用于表示“is-a”关系，接口用于表示“can-do”能力（如 Runnable）。

12. 多线程的实现方式有哪些？

答案：

· 继承 Thread 类，重写 run()。
· 实现 Runnable 接口，传入 Thread。
· 实现 Callable 接口 + FutureTask（可返回结果、抛出异常）。
· 通过线程池（ExecutorService）创建。

13. sleep() 和 wait() 的区别？

答案：

· sleep() 属于 Thread 类，wait() 属于 Object 类。
· sleep() 不会释放锁，wait() 会释放锁并进入等待队列。
· sleep() 时间到自动唤醒，wait() 需要 notify()/notifyAll() 唤醒。
· sleep() 可在任何地方调用，wait() 必须在 synchronized 块/方法中。

14. break、continue、return 的区别？

答案：

· break：跳出当前循环（for、while）或 switch 语句。
· continue：结束本次循环，进入下一次迭代。
· return：结束当前方法，可返回值。

15. 数组和集合的区别？

答案：

· 数组长度固定，集合长度可变。
· 数组可存储基本类型和引用类型，集合只能存储引用类型（通过包装类存储基本类型）。
· 数组提供 length 属性，集合提供 size() 方法。
· 集合提供了更丰富的数据结构和算法（如 List、Set、Map）。

16. 泛型的作用是什么？

答案：
泛型提供编译时类型安全检查，避免强制类型转换。例如 List<String> 只能添加 String，取出时无需转型。泛型在编译后会被擦除（类型擦除），运行时无泛型信息。

17. 自动装箱和拆箱是什么？

答案：
自动装箱：基本类型自动转换为对应的包装类（如 int -> Integer）。
自动拆箱：包装类自动转换为基本类型（如 Integer -> int）。
示例：Integer i = 10; 编译器自动调用 Integer.valueOf(10)。

18. 什么是重载（Overloading）和重写（Overriding）？

答案：

· 重载：同一类中方法名相同，参数列表不同（个数、类型、顺序），与返回值无关，编译期决定。
· 重写：子类重新定义父类的方法，方法签名必须相同，访问权限不能更严格，返回值类型可为子类（协变），运行期决定。

19. public、protected、默认、private 修饰符的作用范围？

答案：

· public：所有类可见。
· protected：同包 + 子类（不同包也可）。
· 默认（无修饰符）：仅同包可见。
· private：仅本类可见。

20. 基本数据类型和引用数据类型的区别？

答案：

· 基本类型：byte、short、int、long、float、double、char、boolean，直接存储值，在栈上分配。
· 引用类型：类、接口、数组、枚举，存储对象的地址（引用），对象在堆上分配。

21. Thread 和 Runnable 的区别？

答案：

· Thread 是类，Runnable 是接口。
· 实现 Runnable 可避免单继承限制，更适合多个线程共享同一任务（如卖票）。
· Thread 本身也实现了 Runnable。

22. synchronized 和 Lock 的区别？

答案：

· synchronized 是 JVM 关键字，自动释放锁；Lock 是 API 接口，需手动 unlock()。
· synchronized 不可中断，Lock 可中断（lockInterruptibly()）。
· synchronized 非公平锁，Lock 可设置公平性。
· Lock 可尝试获取锁（tryLock()）。

23. volatile 关键字的作用？

答案：

· 保证变量的可见性：一个线程修改后，其他线程立即看到最新值。
· 禁止指令重排序。
· 不保证原子性（例如 volatile int count，count++ 仍非线程安全）。

24. 什么是线程池？常见的线程池有哪些？

答案：
线程池复用线程，减少创建销毁开销。常见线程池（Executors 工厂类提供）：

· newFixedThreadPool：固定数量线程，任务队列无界。
· newCachedThreadPool：可缓存线程，空闲 60s 回收。
· newSingleThreadExecutor：单线程。
· newScheduledThreadPool：支持定时/周期性任务。
  实际开发建议使用 ThreadPoolExecutor 直接创建，避免 OOM 风险。

25. JVM 的类加载机制是怎样的？

答案：

· 加载：查找并加载类的二进制数据。
· 验证：确保字节码正确性。
· 准备：为静态变量分配内存并赋默认零值。
· 解析：将符号引用转为直接引用。
· 初始化：执行静态代码块和静态变量赋值。
  类加载器：Bootstrap（根）、Extension（扩展）、Application（应用），双亲委派模型防止重复加载和核心类篡改。

26. Redis 是单线程的吗？为什么这么快？

答案：
Redis 6.0 之前核心网络模型是单线程，6.0 后引入多线程处理网络 I/O 读写，但命令执行仍是单线程。快速原因：

· 基于内存操作。
· 高效数据结构（跳表、哈希表等）。
· 单线程避免上下文切换和锁竞争。
· 非阻塞 I/O 多路复用（epoll）。

27. Redis 支持哪些基本数据结构？

答案：
String（字符串）、Hash（哈希）、List（列表）、Set（集合）、Sorted Set（有序集合，Zset）。以及 Bitmap、HyperLogLog、Geo、Stream 等扩展结构。

28. RDB 与 AOF 的区别？

答案：

· RDB：生成内存快照，文件紧凑，恢复快，但可能丢失最后一次快照后的数据。
· AOF：记录写命令，数据更完整（可配置每秒同步），但文件体积大，恢复慢。
· 可混合使用（Redis 4.0 后）：RDB 做全量，AOF 做增量。

29. RDB 持久化的优缺点？

答案：
优点：

· 单个文件，适合备份和灾难恢复。
· 恢复大数据量时比 AOF 快。
· 对性能影响小（fork 子进程完成）。
  缺点：
· 无法做到实时持久化，可能丢失数据。
· 数据量大时 fork 可能耗时。

30. AOF 日志的三种同步策略？

答案：

· always：每个命令都同步，最安全但性能最差。
· everysec：每秒同步一次，最多丢失 1 秒数据，性能与安全折中。
· no：由操作系统决定何时同步，性能最好但可能丢失较多数据。

31. Redis 的内存淘汰策略有哪些？

答案：

· noeviction：内存不足时写操作报错。
· allkeys-lru：所有 key 中淘汰最近最少使用的。
· volatile-lru：仅设置了过期时间的 key 中淘汰 LRU。
· allkeys-random：随机淘汰。
· volatile-random：仅有过期时间的 key 随机淘汰。
· volatile-ttl：淘汰剩余生存时间（TTL）最短的 key。
· allkeys-lfu（4.0+）：所有 key 淘汰最不经常使用的。
· volatile-lfu（4.0+）：有过期时间的 key 淘汰 LFU。

32. 如何避免缓存穿透、击穿、雪崩？

答案：

· 穿透：查询不存在的数据 → 布隆过滤器、缓存空对象（短过期时间）。
· 击穿：热点 key 过期 → 互斥锁（setnx）、逻辑过期、永不过期+后台更新。
· 雪崩：大量 key 同时过期 → 过期时间加随机值、多级缓存、熔断降级。

33. Redis 的过期删除策略是什么？

答案：

· 惰性删除：访问时检查过期，过期则删除。
· 定期删除：每 100ms 随机抽查一批 key，删除过期的。
· 两种结合，不会使用定时器。

34. Redis 分布式锁如何实现？

答案：
常用 SET key value NX EX seconds（原子操作）。

· 加锁：SET lock_key unique_id NX PX 30000
· 解锁：使用 Lua 脚本判断 value 是否为自己再删除，防止误删。
· 高级方案：Redlock（多节点），但有争议，简单场景用单节点即可。

35. Redis 主从复制的原理？

答案：

· 从节点执行 slaveof，保存主节点信息。
· 从节点与主节点建立 socket 连接。
· 主节点 fork 子进程生成 RDB，发送给从节点，期间写命令缓存到 repl_backlog。
· 从节点清空旧数据，加载 RDB，再执行 backlog 中的命令。
· 之后主节点将写命令实时同步给从节点（异步复制）。

36. Redis 哨兵模式的作用？

答案：

· 监控主从节点健康状态。
· 自动故障转移：主节点下线后，从节点升级为主节点。
· 配置中心：客户端通过哨兵获取当前主节点地址。
· 哨兵本身支持集群，高可用。

37. Redis Cluster 集群的工作原理？

答案：

· 哈希槽分区（16384 个槽），每个节点负责一部分槽。
· 客户端发送命令，节点计算 key 的 CRC16 值对 16384 取模，若不在本节点则返回 MOVED 重定向。
· 集群节点间通过 Gossip 协议通信。
· 支持主从复制，主节点故障时从节点自动提升。

38. 缓存与数据库双写一致性如何保证？

答案：
常见策略：

· 先更新数据库，再删除缓存（推荐）。
· 延迟双删：先删缓存，再更新数据库，再异步延迟删一次。
· 订阅 binlog（如 Canal）异步更新缓存。
· 强一致性场景使用分布式锁，但性能差。

39. 什么是 BigKey？如何发现和处理？

答案：
BigKey 指单个 key 存储的数据过大（如大的 hash、list、zset）。
发现：redis-cli --bigkeys、SCAN + DEBUG OBJECT。
处理：

· 拆分：如 hash 拆成多个小 hash。
· 删除：使用 UNLINK（非阻塞）而非 DEL。
· 避免一次性读取全部数据。

40. Redis 的 Pipeline 有什么作用？

答案：
Pipeline 将多个命令打包一次发送给 Redis，减少网络往返时间（RTT），提高吞吐量。但命令仍然按顺序执行，不具备原子性。

41. InnoDB 与 MyISAM 的核心区别？

答案：

特性 InnoDB MyISAM
事务 支持 不支持
外键 支持 不支持
行锁/表锁 行锁（默认） 表锁
MVCC 支持 不支持
崩溃恢复 好（redo log） 差
全文索引 5.6后支持 支持

42. InnoDB 的 MVCC 实现原理？

答案：
MVCC（多版本并发控制）通过隐藏字段（DB_TRX_ID、DB_ROLL_PTR）和 undo log 实现。每个事务启动时生成 Read View（活跃事务 ID 列表）。查询时只看到事务 ID 小于当前且未提交的事务版本，实现非阻塞读。

43. B+树索引与哈希索引的区别？

答案：

· B+树：有序，支持范围查询、前缀查找、排序；适合磁盘 I/O。
· 哈希：无序，仅支持等值查询，查询速度 O(1)，但不支持范围查询，存在哈希冲突。

44. 事务的四大特性（ACID）是什么？

答案：

· 原子性：事务中操作要么全做，要么全不做（undo log）。
· 一致性：事务前后数据完整性约束不被破坏。
· 隔离性：并发事务互不干扰（锁+MVCC）。
· 持久性：事务提交后数据永久保存（redo log）。

45. 什么是事务的隔离级别？MySQL 默认是哪个？

答案：

· 读未提交（RU）：可能脏读。
· 读已提交（RC）：避免脏读，可能出现不可重复读。
· 可重复读（RR）：MySQL 默认，避免脏读和不可重复读，但可能幻读（InnoDB 通过间隙锁避免了幻读）。
· 串行化：强制事务串行，性能最低。

46. InnoDB 的行锁和表锁有什么区别？

答案：

· 行锁：粒度细，并发高，但开销大，可能出现死锁。
· 表锁：粒度粗，并发低，开销小，不会死锁（但意向锁会）。
  InnoDB 默认行锁，但更新未使用索引时会升级为表锁。

47. 什么是死锁？如何避免？

答案：
死锁是两个或多个事务相互等待对方释放锁。
避免：

· 固定访问顺序。
· 使用 SELECT ... FOR UPDATE 时按主键顺序加锁。
· 合理使用索引，减少锁范围。
· 设置锁等待超时（innodb_lock_wait_timeout）。

48. 聚簇索引和非聚簇索引的区别？

答案：

· 聚簇索引：叶子节点存储完整行数据，每个表只能有一个（通常是主键）。
· 非聚簇索引（二级索引）：叶子节点存储主键值，需回表查询完整数据。

49. 如何定位和优化慢查询？

答案：

1. 开启慢查询日志（slow_query_log），设置阈值（long_query_time）。
2. 使用 mysqldumpslow 或 pt-query-digest 分析。
3. 对慢 SQL 执行 EXPLAIN 查看执行计划。
4. 优化：加索引、重写 SQL（避免函数、隐式转换）、分页优化、避免 SELECT *。

50. EXPLAIN 命令的关键字段有哪些？

答案：

· type：访问类型（system > const > eq_ref > ref > range > index > ALL，至少到 range）。
· possible_keys：可能用到的索引。
· key：实际使用的索引。
· rows：预估扫描行数。
· Extra：额外信息（Using index 表示覆盖索引，Using filesort 需要优化）。

51. 索引失效的常见场景有哪些？

答案：

· 使用 != 或 <>。
· 使用 LIKE '%xxx'（左模糊）。
· 对索引列进行函数操作（如 WHERE DATE(create_time) = '2024-01-01'）。
· 隐式类型转换（如字符串列与数字比较）。
· OR 条件中有一侧无索引。
· 复合索引未使用最左前缀。

52. 什么是覆盖索引？

答案：
一个索引包含所有需要查询的字段，无需回表。例如 SELECT id, name FROM user WHERE name = '张三'，若 (name, id) 联合索引覆盖，则直接返回。

53. 什么是回表查询？

答案：
使用非聚簇索引查询时，索引叶子节点存储的是主键值，需要再根据主键到聚簇索引中查找完整行数据，这个过程称为回表。

54. 前缀索引的作用是什么？

答案：
对于字符串列，只索引前几个字符，减少索引大小，提高查询效率。例如 ALTER TABLE user ADD INDEX idx_name(name(10))。缺点是可能降低区分度，且不能用于 ORDER BY 或 GROUP BY。

55. 主键为什么推荐使用自增 ID 而不是 UUID？

答案：

· 自增 ID 是顺序插入，B+树分裂少，页利用率高。
· UUID 随机无序，插入时频繁页分裂，性能差，且占用空间大（16字节 vs 4/8字节）。
· 分布式场景可使用雪花算法。

56. 分库分表的策略有哪些？

答案：

· 垂直分库：按业务拆分（订单库、用户库）。
· 垂直分表：按列拆分（将大字段或访问频率低的列分到扩展表）。
· 水平分表：按哈希、范围、列表等分片键将数据分散到多张表。
· 常用中间件：ShardingSphere、MyCat。

57. 读写分离如何实现？主从延迟问题如何解决？

答案：

· 实现：主库处理写，从库处理读，通过中间件（如 ShardingSphere）或代理（如 ProxySQL）路由。
· 主从延迟原因：从库回放 binlog 慢。
· 解决方案：
  · 强制读主（重要场景）。
  · 延迟容忍：缓存标记写时间，从库读取时等待。
  · 并行复制（MySQL 5.7+ MTS）。
  · 业务层面避免写入后立即读。

58. INNER JOIN、LEFT JOIN、RIGHT JOIN 的区别？

答案：

· INNER JOIN：只返回匹配的行。
· LEFT JOIN：返回左表所有行，右表不匹配则为 NULL。
· RIGHT JOIN：返回右表所有行，左表不匹配则为 NULL。

59. UNION 和 UNION ALL 的区别？

答案：

· UNION：合并结果集并去重，有排序开销。
· UNION ALL：直接合并，不去重，性能更高。

60. COUNT(*)、COUNT(1)、COUNT(列) 的区别？

答案：

· COUNT(*) 和 COUNT(1) 都统计所有行数，包括 NULL，性能几乎相同。
· COUNT(列) 统计该列非 NULL 值的行数。
· InnoDB 中 COUNT(*) 无 where 时需扫全表，MyISAM 有单独计数。

61. CHAR 和 VARCHAR 的区别？

答案：

· CHAR：定长，长度范围 0~255，存储时用空格填充到指定长度，检索时去除尾部空格。适合存储长度固定的数据（如身份证号、手机号）。
· VARCHAR：变长，长度范围 0~65535，存储时额外 1~2 字节记录长度，节省空间但可能产生碎片。适合长度变化较大的数据（如地址、描述）。

62. 什么是视图？什么是存储过程？

答案：

· 视图：虚拟表，基于 SQL 查询结果，不存储数据，用于简化复杂查询或控制数据访问权限。
· 存储过程：预编译的 SQL 代码块，可包含流程控制、参数输入输出，减少网络传输，提高执行效率。

63. 什么是触发器？

答案：
触发器是一种特殊的存储过程，在某个表上执行 INSERT、UPDATE、DELETE 操作时自动触发执行。用于实现数据审计、约束检查、级联更新等。注意过度使用会增加数据库维护难度。

64. 如何备份和恢复 MySQL 数据库？

答案：

· 逻辑备份：mysqldump -u root -p dbname > backup.sql，恢复：mysql -u root -p dbname < backup.sql。
· 物理备份：直接复制数据文件（如使用 xtrabackup），适合大数据量。
· 增量备份：基于 binlog 或 xtrabackup 的增量功能。

65. MySQL 日志文件有哪些（binlog、redo log、undo log）？分别的作用是什么？

答案：

· binlog（归档日志）：记录所有修改数据库的 SQL 语句或行变更，用于主从复制和基于时间点的恢复。
· redo log（重做日志）：InnoDB 特有，保证持久性。记录事务对数据页的物理修改，崩溃恢复时重做。
· undo log（回滚日志）：记录事务修改前的旧值，用于回滚和 MVCC。

66. 进程和线程的区别？

答案：

· 进程是资源分配的最小单位，线程是 CPU 调度的最小单位。
· 进程拥有独立地址空间，线程共享进程的地址空间和资源。
· 进程间通信（IPC）复杂（管道、消息队列、共享内存等），线程间通信简单（共享变量）。
· 进程创建和切换开销大，线程开销小。

67. 进程间通信（IPC）的方式有哪些？

答案：

· 管道（Pipe） / 命名管道（FIFO）
· 消息队列
· 共享内存（最快，但需同步）
· 信号量（用于同步）
· 信号（Signal）
· Socket（可用于不同机器）

68. 什么是死锁？死锁产生的必要条件有哪些？

答案：
死锁是两个或多个进程互相等待对方释放资源，导致无法继续执行。
四个必要条件：

1. 互斥：资源一次只能被一个进程使用。
2. 持有并等待：进程持有资源同时等待其他资源。
3. 不可剥夺：已分配的资源不能强行剥夺。
4. 循环等待：形成等待环路。

69. 什么是虚拟存储器？

答案：
虚拟存储器将物理内存和磁盘存储结合，为每个进程提供连续的、比物理内存更大的地址空间。通过页表将虚拟地址映射到物理地址或磁盘，支持多进程并发，隔离地址空间。

70. 分页和分段的区别？

答案：

· 分页：将内存和进程空间划分为固定大小的页，地址空间连续，无外碎片，但可能有内碎片。用户不可见。
· 分段：按逻辑模块（如代码段、数据段）划分，段大小可变，有外碎片，用户可见。便于共享和保护。

71. 什么是系统调用？

答案：
系统调用是操作系统提供给应用程序的接口，用于请求内核服务（如文件操作、进程控制、网络通信）。用户态程序通过软中断或指令陷入内核态执行。

72. 什么是用户态和内核态？

答案：

· 用户态：CPU 只能执行非特权指令，访问受限的内存区域，应用程序运行在此模式。
· 内核态：CPU 可执行所有指令，访问所有内存，操作系统内核运行在此模式。
  切换通过系统调用、中断、异常实现，开销较大。

73. 什么是并行？什么是并发？

答案：

· 并发：多个任务在单核 CPU 上交替执行（宏观同时，微观串行）。
· 并行：多个任务在多核 CPU 上同时执行。
  并发是逻辑上的同时，并行是物理上的同时。

74. 进程调度算法有哪些？

答案：

· 先来先服务（FCFS）
· 短作业优先（SJF）
· 优先级调度
· 时间片轮转（RR）
· 多级反馈队列（MLFQ，综合多种策略）

75. 什么是中断？

答案：
中断是 CPU 对系统发生的异步事件的响应。中断发生后，CPU 暂停当前程序，保存现场，跳转到中断处理程序执行，完成后恢复。包括硬件中断（I/O 设备）和软件中断（异常、系统调用）。

76. 什么是缓冲区溢出？

答案：
缓冲区溢出是指向缓冲区写入数据时超出其边界，覆盖相邻内存。可能导致程序崩溃或恶意代码执行。常见于 C/C++ 的不安全函数（如 strcpy、gets）。防范措施：使用安全函数、栈保护（Canary）、ASLR、DEP。

77. 什么是内存泄漏？

答案：
内存泄漏是指程序动态分配内存后，不再使用但未释放，导致可用内存逐渐减少。长期运行可能导致系统崩溃。Java 等有 GC 的语言仍可能发生（如集合中无用的对象引用未清除）。

78. 什么是虚拟地址空间？

答案：
虚拟地址空间是每个进程可见的线性地址集合（如 32 位系统 4GB）。操作系统通过 MMU 将其映射到物理内存或磁盘。优点：隔离进程、简化内存管理、提供更大地址空间。

79. 什么是页面置换算法？

答案：
当物理内存不足时，操作系统将某些页换出到磁盘。常见置换算法：

· 最佳置换（OPT，理想）
· 最近最久未使用（LRU）
· 先进先出（FIFO，有 Belady 异常）
· 时钟算法（Clock）
· 最不常用（LFU）

80. 什么是孤儿进程和僵尸进程？

答案：

· 孤儿进程：父进程先于子进程结束，子进程被 init（PID=1）收养，没有危害。
· 僵尸进程：子进程结束，父进程未调用 wait() 回收其 PCB，导致进程表中残留条目。大量僵尸进程会耗尽进程 ID。解决方法：杀死父进程或父进程调用 wait()。
