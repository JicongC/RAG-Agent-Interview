# 基于 RAG 与 Agent 的智能面试辅导系统

**项目时间：** 2026.01–2026.03  
**技术栈：** Python、LangChain/LangGraph、ChromaDB、Text-Embedding-v4、GTE-Rerank-v2、Qwen2.5、QLoRA、PEFT、Transformers、Streamlit

## 简历推荐版

- 实现知识问答与模拟面试双模式，支持连续追问、面试报告生成和按用户 ID 持久化会话状态，并集成 RAG 检索、天气、城市及用户信息等 4 类 Agent 工具。
- 使用 `RecursiveCharacterTextSplitter` 完成文档切分与向量入库，通过文件 MD5 实现增量更新与重复文档过滤。
- 采用 Text-Embedding-v4 进行语义检索，召回 Top-12 候选片段后使用 GTE-Rerank-v2 重排，并选取 Top-3 注入上下文；重排服务异常时自动降级为向量检索结果。
- 基于 Qwen2.5-1.5B-Instruct 搭建面试问答 QLoRA 微调实验，构建 1,200 条指令数据并按 8:1:1 划分训练集、验证集和测试集；配置 LoRA Rank 16、Alpha 32、学习率 2e-4 和 3 Epoch，支持微调前后自动指标与人工评分对比。
- 实现训练实验追踪，自动记录 GPU 型号、总显存、PyTorch/NVML 峰值显存、训练耗时、训练损失及验证损失，并保存最佳 LoRA Adapter 和实验配置。
- 在 RTX 4060 Laptop 8GB 上完成 QLoRA 实验，训练耗时 21.28 分钟、NVML 峰值显存约 4.50GB；通过验证 Loss 识别过拟合，并自动回载 Step 50 的最佳 LoRA 检查点。
- 基于配置文件与环境变量统一管理模型、向量库和外部 API 参数，完成 3 个用户状态样例及多轮面试记录的持久化验证。

## 更精简的四点版

- 基于 LangChain/LangGraph 构建 RAG + Agent 面试辅导系统，实现知识问答、模拟面试、连续追问、报告生成及用户会话持久化，并集成 4 类 Agent 工具。
- 使用 ChromaDB、Text-Embedding-v4 和 GTE-Rerank-v2 搭建“Top-12 召回、Top-3 重排”检索链路，支持 MD5 增量入库、重复过滤和重排异常自动降级。
- 基于 Qwen2.5-1.5B-Instruct 实现 QLoRA 面试问答微调流程，构建 1,200 条指令数据并按 8:1:1 划分，采用 Rank 16、Alpha 32、学习率 2e-4、3 Epoch 完成实验配置。
- 实现微调前后回答质量评测与训练资源追踪，支持字符 F1、ROUGE-L、人工多维评分及 GPU 峰值显存、训练时间和验证 Loss 自动记录。
